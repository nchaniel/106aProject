ME C106A · Final Engineering Report

Vision-Guided Pick and Place
with a UR7e Robot Arm

A closed-loop robotic system that detects food objects on a table with a wrist-mounted Intel RealSense camera, reconstructs their 3D positions in the robot's base frame, and executes MoveIt 2-planned grasps that deposit each object onto a detected plate.

1 System Overview

Two phases. An offline inspection orbit builds a multi-view dataset of the scene; an online executor turns the resulting 6D positions into a sequence of grasps. Each phase below is a brief — full technical content lives in the interactive architecture diagram.

01 Inspection

The arm orbits the plate in two rows of twenty waypoints, with the camera permanently aimed at the centroid. Each waypoint saves an RGB capture and the corresponding end-effector pose. When forty frames are written, /orbit_done latches.

02 Execution

The commander runs YOLO + SAM2 segmentation and multi-view triangulation to produce object_poses.npy. The user presses Enter and UR7e_CubeGrasp executes a seven-step grasp for each object, returning home between picks.

5 Status & What's Next

Perception and orbit inspection are stable, the offline 6D pipeline runs end-to-end, and pick execution works on rigid objects. Placement on the plate is the next milestone.

Working

  • Live YOLOv8 detection + plate-radius exclusion
  • 40-waypoint orbit with synchronized pose logging
  • SAM2 batch segmentation across all captures
  • Multi-view RANSAC triangulation in base_link
  • Async MoveIt-planned pick sequence with refinement

In progress

  • Wire position_base_link_m into main.py.
  • Per-object drop spots around the plate centroid.
  • Pick ordering by distance / class priority.

Known issues

  • Drop-waypoint IK failures not yet checked before enqueue.
  • Gripper state silently drifts if it starts closed.
  • Refined detection waits indefinitely if the object disappears.
  • colcon build copies (not symlinks) Python files.
