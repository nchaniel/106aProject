Robot Ratatouille: Culinary Robot Project

Culinary plating assistant made with an Intel RealSense camera paired with a UR7e Robotic Arm. Detects, scans, and memorizes plated dishes to replicate on command.

1 System Overview

There are two phases. An offline inspection orbit builds a multi-view dataset of the scene using MoveIt, YOLO, and SAM2; an online executor turns the resulting 6D positions into a sequence of grasps. Each phase below is a brief — more technical content is in the interactive architecture diagram.

01 Inspection

The arm orbits the plate in two rows of twenty waypoints, with the end-effector tool permanently aimed at the centroid (except for the overhead shot). Each waypoint saves an RGB capture as a .png file and the corresponding end-effector pose into an .npy file. When forty one frames are written, /orbit_done becomes True and segmentation begins.

02 Execution

The commander runs YOLO + SAM2 segmentation and Weighted DLT triangulation to produce object_poses.npy, which contains the info about the estimate positions of each detected object. The user presses Enter and UR7e_CubeGrasp executes a seven-step grasp for each object, returning home between picks.

4 Status & What's Next

Perception and orbit inspection are stable, the offline 6D pipeline runs end-to-end, and pick execution works on rigid objects. Placement on the plate is the next milestone.

Working

  • Live YOLOv8 detection + plate-radius exclusion
  • 41-waypoint orbit with synchronized pose logging
  • SAM2 batch segmentation across all captures
  • Multi-view RANSAC triangulation in base_link
  • Async MoveIt-planned pick sequence with refinement

In progress

  • Wire position_base_link_m into main.py.
  • Per-object drop spots around the plate centroid.
  • Pick ordering by distance / class priority.

Known issues

  • Drop-waypoint IK failures not yet checked before enqueue.
  • Gripper state silently drifts if it starts closed.
  • Refined detection waits indefinitely if the object disappears.
  • colcon build copies (not symlinks) Python files.

6 AI Use Disclosure

Throughout the project creation process, ChatGPT, Gemini, and Claude Code were used in supporting roles for debugging, code integration, logic design, and website development.

Bug Fixes

Although we were able to independently resolve most issues, several sections of the codebase contained logical errors, particularly during integration of the UR7e arm. These issues often resulted in unintended robot behavior or runtime errors that prevented execution. AI was used to suggest edits, including adding, removing, and modifying lines of code to resolve these bugs efficiently.

Code Integration

The project was developed in modular components (arm circling, bounding boxes, pixel masking, etc), allowing parallel development across team members. However, while each module functioned independently, integrating them into a unified system often resulted in code that would often fail after minor adjustments. AI was used to consolidate these components into a cohesive pipeline, including merging scripts and restructuring packages, to allow the packagers to work together.

Logic and Implementation Support

Although the intended system behavior was well understood, AI was used to assist in translating high-level ideas into implementation details. This included generating pseudocode, as well as adjusting logic as necessary to allow the pipeline to work as intended.

Website Development

The formatting, layout, and visual design of this website were assisted using AI tools. However, all content—test descriptions, images, and images— were manually created and curated.
